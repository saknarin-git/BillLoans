# แผนเปลี่ยนระบบสมัครใช้งานแบบรออนุมัติ

## เป้าหมาย
ปรับหน้าเข้าสู่ระบบ/สมัครใช้งานให้ใช้รูปแบบใหม่ดังนี้
- เก็บข้อมูลสมัคร: คำนำหน้าชื่อ, ชื่อ, สกุล, Username, Password
- Username ต้องไม่ซ้ำกัน
- Password ต้องถูกเก็บแบบ Hash/Encrypted record
- หลังสมัคร บัญชีต้องอยู่สถานะรออนุมัติ
- ผู้ดูแลระบบต้องอนุมัติก่อน จึงจะเข้าสู่ระบบได้
- ลดการพึ่งพา Google Apps Script สำหรับ flow สมัคร/ล็อกอิน/อนุมัติผู้ใช้

## สิ่งที่ปรับแล้วในรอบนี้
1. หน้า Auth เปลี่ยน UX เป็น approval-first
2. ฟอร์มสมัครลดเหลือข้อมูลสำคัญตามรูปแบบใหม่
3. สมัครสำเร็จแล้วกลับไปหน้าเข้าสู่ระบบพร้อมข้อความรออนุมัติ
4. Backend ฝั่ง Supabase Edge รองรับ
   - verifyLogin
   - registerUser
   - updateCurrentUserProfile
   - adminUpdateUserAccount
5. การเรียก backend หลักของหน้าเว็บเปลี่ยนเป็น Edge-first แล้วค่อย fallback

## Flow ใหม่
1. ผู้ใช้กรอกคำนำหน้าชื่อ / ชื่อ / สกุล / Username / Password
2. ระบบตรวจสอบว่า Username ซ้ำหรือไม่
3. ระบบ hash รหัสผ่านก่อนบันทึก
4. บัญชีถูกสร้างด้วยสถานะ Pending
5. ผู้ดูแลระบบเข้าเมนูจัดการผู้ใช้เพื่ออนุมัติ
6. เมื่อสถานะเป็น Active แล้ว ผู้ใช้จึงล็อกอินได้

## จุดตรวจรับงาน
- [ ] สมัครผู้ใช้ใหม่ได้
- [ ] Username ซ้ำแล้วถูกปฏิเสธ
- [ ] บัญชี Pending ยังล็อกอินไม่ได้
- [ ] Admin เปลี่ยนสถานะเป็น Active ได้
- [ ] บัญชี Active ล็อกอินได้
- [ ] หน้า Auth ไม่บังคับยืนยันอีเมลก่อนใช้งาน
- [ ] หน้าเว็บเรียก backend ผ่าน Supabase Edge ได้เป็นหลัก

## ขั้นตอนถัดไปที่แนะนำ
- เพิ่มปุ่มอนุมัติ/ระงับแบบชัดเจนในตารางผู้ใช้
- ย้ายฟังก์ชันลืมรหัสผ่านออกจาก GAS ให้ครบ
- ปิด fallback ไป GAS เมื่อ Edge ครบทุกเมธอดที่จำเป็น
